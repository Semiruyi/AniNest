# 元数据识别策略设计文档

## 1. 目标与范围

### 1.1 目标
将用户本地文件夹/文件名中**非结构化的文本**，转换为 Bangumi API 可识别的**结构化搜索关键词**，并在返回结果中**准确匹配**到正确的番剧条目。

### 1.2 范围
- **输入**：文件夹名（主要来源）、视频文件名（辅助来源）。
- **输出**：Bangumi `subject_id`（或确认无匹配）。
- **限制**：面向动画/番剧；暂不支持电影、美剧、纪录片。

---

## 2. 输入来源与优先级

### 2.1 优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| **P1** | 文件夹名 | 绝大多数用户按番剧名整理文件夹，信息最干净。 |
| **P2** | 父文件夹名 | 若文件夹名为"Season 1"、"01-25"等无意义名称，回退到父级。 |
| **P3** | 视频文件名 | 若整个路径均无有效文件夹名，取第一个视频文件名。 |

### 2.2 决策逻辑

```
keyword = ExtractFolderName(folderPath)
if IsMeaningless(keyword):
    keyword = ExtractParentFolderName(folderPath)
if IsMeaningless(keyword):
    keyword = ExtractFromFirstVideoFile(folderPath)
```

**判定"无意义"的标准**：
- 纯数字（如 `01`、`1080p`）。
- 常见季数标识（`Season 1`、`S01`、`第1季`、`TV`）。
- 常见集数范围（`01-25`、`E01-E25`、`全25集`）。
- 清洗后长度小于 2 个字符。

---

## 3. 清洗策略（Noise Reduction）

### 3.1 清洗流水线

采用**多阶段流水线**，每阶段解决一类噪音：

```
原始文本
  → 阶段1：去除包围标记（方括号、圆括号、尖括号）
  → 阶段2：去除技术标签（分辨率、编码、音轨）
  → 阶段3：去除发布信息（字幕组、种子站、日期戳）
  → 阶段4：去除集数/季数标识
  → 阶段5：去除无意义介词/符号
  → 阶段6：标准化（全角转半角、去除多余空格）
  → 输出关键词
```

### 3.2 阶段详解

#### 阶段1：去除包围标记

用户命名常用方括号/圆括号包裹元信息，需要识别并剔除这些标记块：

| 模式 | 示例 | 处理后 |
|------|------|--------|
| `[...]` | `[GM-Team][进击的巨人]` | `进击的巨人` |
| `(...)` | `(1080p)(x265)` | 空（若仅剩技术标签则继续阶段2） |
| `【...】` | `【喵萌奶茶屋】` | 空 |
| `〖...〗` | `〖BDrip〗` | 空 |
| `<...>` | `<NC-Raws>` | 空 |

**规则**：
- 若标记块内**不含任何中文字符或日文假名**，视为纯元信息标记，直接剔除。
- 若标记块内**含有中文/日文**，可能是标题本身（如 `[进击的巨人]`），保留内部文本进入下一阶段。

#### 阶段2：去除技术标签

| 类别 | 正则/关键词示例 | 说明 |
|------|----------------|------|
| 分辨率 | `1080p`, `720p`, `4K`, `2160p`, `1920x1080` | 大小写不敏感 |
| 视频编码 | `x264`, `x265`, `HEVC`, `AVC`, `H264`, `H265`, `10bit` | |
| 音频格式 | `AAC`, `FLAC`, `AC3`, `DTS`, `5.1ch` | |
| 片源 | `BDRip`, `WEBRip`, `HDTV`, `BluRay`, `DVDRip`, `UHD`, `Remux` | |
| 文件格式 | `.mp4`, `.mkv`, `.avi` | 去除扩展名 |
| 字幕 | `GB`, `BIG5`, `CHS`, `CHT`, `简日`, `繁日` | |
| 其他技术 | `V2`, `fix`, `v2`, `repack` | 版本修正标识 |

#### 阶段3：去除发布信息

| 类别 | 示例 | 说明 |
|------|------|------|
| 字幕组名 | `喵萌奶茶屋`, `DMG`, `天香字幕社`, `幻之字幕组` | 较难穷尽，采用启发式规则 |
| 种子站标识 | `动漫花园`, `acg.rip`, `nyaa` | |
| 日期戳 | `2023.01`, `20230115` | 4位或6位或8位数字组合 |
| CRC32 | `[A1B2C3D4]` | 8位十六进制在方括号内 |

**字幕组名启发式检测**：
- 位于最外层方括号/圆括号内。
- 不含任何常见动画标题关键词（如"的"、"之"、"物语"等）。
- 长度通常在 2-8 个字符之间（中文）或包含英文字母缩写。
- 常见后缀：`字幕组`, `字幕社`, `汉化组`, `压制组`。

#### 阶段4：去除集数/季数标识

| 类型 | 正则示例 | 说明 |
|------|---------|------|
| 集数范围 | `01-25`, `E01-E25`, `第01-25话`, `全25话`, `1-25集` | |
| 单集 | `第01话`, `EP01`, `E01`, `01` | 若上下文判断为单集而非标题 |
| 季数 | `Season 1`, `S01`, `第1季`, `S1`, `II`, `III`, `2nd` | |
| 卷数 | `Vol.1`, `Vol 1`, `第1卷` | |
| SP/OVA | `SP`, `OVA`, `OAD`, `特典`, `Bonus` | 通常保留，因为可能是标题一部分 |

**特殊处理**：
- `SP` / `OVA` **不直接删除**，因为有些番剧标题确实包含这些词（如"某科学的超电磁炮S"中的 S 不是季数）。仅当紧接在标题后作为后缀时才删除。
- `第二季`、`2期` 等**明确季数标识**必须删除，否则 Bangumi 搜索会匹配到第一季。

#### 阶段5：去除无意义介词与符号

- 开头/结尾的 `-`, `_`, `~`, `·`, `・`, `「`, `」`, `『`, `』`。
- 多余空格、制表符、换行符。

#### 阶段6：标准化

| 操作 | 示例 |
|------|------|
| 全角字母/数字转半角 | `ＡＢＣ` → `ABC`, `１２３` → `123` |
| 全角空格转半角空格 | `　` → ` ` |
| 连续空格压缩为单个 | `进击  的  巨人` → `进击的巨人` |
| 去除首尾空格 | ` 巨人 ` → `巨人` |

### 3.3 清洗示例

| 原始文件夹名 | 阶段处理后 | 说明 |
|-------------|-----------|------|
| `[GM-Team][进击的巨人][01-25][1080p][x265]` | `进击的巨人` | 标准字幕组格式，清洗后理想。 |
| `【喵萌奶茶屋】孤独摇滚！第2季 [1080p][AAC]` | `孤独摇滚` | `！`保留，`第2季`删除。注意 Bangumi 中标题是`孤独摇滚！`不含`第2季`。 |
| `(2023.10) 葬送的芙莉莲 01-28 BDRip` | `葬送的芙莉莲` | 日期戳、集数、片源均去除。 |
| `Season 1` | *回退到父文件夹* | 无意义，触发 P2 回退。 |
| `鬼灭之刃 游郭篇 Vol.1-11` | `鬼灭之刃 游郭篇` | 卷数去除，保留篇章名。 |
| `[NC-Raws] 咒术回战 怀玉·玉折 / 涩谷事变` | `咒术回战 怀玉·玉折 / 涩谷事变` | 字幕组去除，双标题保留。 |
| `EVANGELION_3.0+1.0_THRICE_UPON_A_TIME` | `EVANGELION 3.0+1.0 THRICE UPON A TIME` | 下划线转空格。 |
| `IDOLM@STER` | `IDOLM@STER` | 特殊符号 `@` 是标题一部分，保留。 |

## 4. 匹配与排序策略

### 4.1 Bangumi 搜索接口行为

`POST /v0/search/subjects` 的 `sort=match` 返回按相关度排序的结果列表。Bangumi 的相关度算法基于标题、别名、简介的文本匹配，但**不完美**：

- 同名或高度相似的作品可能混排。
- 剧场版、TV 版、OVA 可能出现在同一结果列表中。
- 部分冷门作品可能没有 `name_cn`，只有日文名。

### 4.2 自动匹配策略（Auto Scrape）

第一阶段**仅支持自动刮削，无手动弹窗**。必须在 API 返回的候选列表中自动决策，且**宁可跳过也不误匹配**。

#### 策略 A：类型过滤（基础）

搜索请求体固定 `filter.type = [2]`，排除书籍、游戏、真人条目。

#### 策略 B：标题相似度验证（核心）

取 Bangumi 返回的 Top1 结果，与清洗后的关键词计算相似度：

```
候选 = Bangumi.Search(keyword).FirstOrDefault()
if 候选 == null: return null

similarity = CalculateSimilarity(keyword, 候选.Title)
if similarity < 0.5:
    return null  // 低置信度，静默跳过
```

**相似度算法**：
- 中文/日文：最长公共子序列（LCS）相似度 = `LCS长度 / Max(关键词长度, 结果标题长度)`。
- 英文：忽略大小写和标点后的 Levenshtein 归一化相似度。

#### 策略 C：基于年份的加权过滤（增强）

若清洗后的关键词中包含**4 位年份**（如 `2023`），优先匹配该年份附近的作品：

```
if keyword 包含年份 year:
    若 abs(候选.Date.Year - year) > 1:
        置信度降级（可接受但不强依赖）
```

年份不匹配**不直接拒绝**（因为部分番剧的放送年份和文件夹命名年份可能因季番跨年度而有偏差），但可作为日志标记。

#### 策略 D：短词歧义拒绝

清洗后的关键词过短时，直接拒绝自动匹配，避免"Fate"、"巨人"等高度歧义词误匹配：

```
if 中文长度 < 3 或 英文长度 < 6:
    return null
```

## 5. 歧义处理

### 5.1 同名不同季

**问题**：`进击的巨人` 在 Bangumi 上有 TV 第 1 季、第 2 季、第 3 季 Part.1、Part.2、最终季 Part.1、Part.2、Part.3 等多个条目。

**策略**：
1. **关键词包含季数**：如 `进击的巨人 第三季`，清洗后保留`第三季`，Bangumi 搜索本身会将其排前列。
2. **文件夹名不含季数**：如 `进击的巨人`，则自动取 Bangumi 最早的一条（通常是第一季）。本阶段无手动纠正入口，若用户需要精确匹配后续季，需后续版本支持手动搜索。
3. **季数关键词映射表**（可选增强）：

| 常见表达 | Bangumi 语义 |
|----------|-------------|
| `第二季`, `2期`, `S2`, `Season 2` | 取第二部作品 |
| `剧场版`, `Movie`, `映画` | 取 `type = 2` 中 `platform = 剧场版` |
| `OVA`, `OAD` | 取 `platform = OVA` |
| `总集篇`, `Recap` | 取总集篇条目 |

### 5.2 同名不同作品

**问题**：`白色相簿` 既指 1998 年的游戏，也指 2009 年的 TV 动画。

**策略**：
- 自动模式下固定 `filter.type = [2]`（动画），排除游戏和书籍。
- 若仍有歧义（如存在 TV 和 OVA 两个动画条目），优先取 TV 版（通常 `platform = "TV"` 排在更前面或集数更多）。

### 5.3 系列作品 vs 单篇

**问题**：文件夹名为`Fate`，里面可能是`Fate/stay night`、`Fate/Zero`、`Fate/Grand Order` 等任何一部。

**策略**：
- `Fate` 清洗后仍然很短（2 个字符），相似度验证会将其标记为低置信度，**不自动采纳**。
- 本阶段静默跳过，不展示任何元数据。

## 6. 置信度与阈值

### 6.1 自动刮削采纳条件

自动刮削不是"有结果就采纳"，需要同时满足以下全部条件：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 搜索返回非空 | `candidates.Count > 0` | |
| Top1 类型为动画 | `type == 2` | 已在请求 filter 中确保 |
| 标题相似度 | `>= 0.5` | 核心门槛，防止误匹配 |
| 非短词歧义 | `keyword.Length >= 3`（中文字符）或 `>= 6`（英文字符） | "AB"、"Fate" 等易歧义词拒绝自动匹配 |
| （可选）年份匹配 | 若关键词含年份，Top1 年份误差 `<= 1` | 辅助验证，不匹配不直接拒绝 |

若任一硬性条件不满足，自动刮削**静默跳过**，不在卡片上展示任何元数据。第一阶段无手动补救入口，跳过即跳过。

### 6.2 置信度分级（第一阶段简化版）

| 等级 | 条件 | 行为 |
|------|------|------|
| **通过** | 相似度 >= 0.5，且非短词歧义 | 自动采纳，后台静默完成，卡片刷新封面。 |
| **拒绝** | 相似度 < 0.5，或短词歧义，或无结果 | 不采纳，不展示，静默跳过。 |

## 7. 回退策略

### 7.1 搜索无结果

```
Bangumi.Search(keyword) 返回空
  → 尝试关键词简化：去除"篇"、"章"、"季"等后缀再搜
    例：`鬼灭之刃 游郭篇` → 搜 `鬼灭之刃`
  → 仍无结果 → 标记为"未找到"，记录日志，不再重试
```

### 7.2 详情获取失败

```
Bangumi.GetDetail(id) 失败（404 或 网络错误）
  → 记录日志
  → 若 404：可能是 Bangumi 数据已删除，标记为"失效"
  → 若网络错误：不标记，下次加载时仍尝试展示已有缓存
```

### 7.3 批量添加文件夹时的策略

`AddFolderBatchAsync` 可能一次性添加数十个文件夹：

- 每个文件夹的刮削任务**串行执行**，间隔 1s，避免并发触发 Bangumi 限流。
- 不阻塞 UI 线程，后台队列逐步完成。

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| V1 | 实现基础清洗（阶段 1-6）+ 严格自动匹配（相似度 >= 0.5 + 短词拒绝 + 类型过滤） | P0 |
| V1.1 | 添加年份加权过滤，提升含年份命名场景的准确率 | P1 |
| V1.2 | 添加季数关键词映射表，减少同名不同季误匹配 | P1 |
| V2 | 引入视频文件名回退（P3 优先级），处理无意义文件夹名场景 | P2 |
| V2.1 | 日文假名标准化（片假名→平假名对比） | P2 |
